# 📘 Kiroを用いたIT用語辞典実装プロジェクト

## 🧠 プロジェクト概要

このプロジェクトは、**AmazonのAI開発支援ツール「Kiro」**を活用して、  
構造化された**IT用語辞典を半自動で作成・管理する仕組み**を構築する試みです。

### 目的

- AIを活用して効率的にIT用語辞典を構築したい
- 一貫性のあるフォーマットで複数用語を自動生成したい
- GitHubベースで履歴管理・コラボレーション可能な辞書を目指す

---

## 🔧 使用ツール・技術

- **Kiro（Claude 3 + Amazon Q）**
- **GitHub（Issue / PR / Actions）**
- **Markdown構成**
- （必要に応じて：CodeCatalyst や Slack連携）

---

## 📐 用語テンプレート（例）

```md
# 🧠 用語名（英語）

## 🔍 定義
ここに簡潔な定義を書く

## 🔧 技術的背景・構成要素
- ○○
- △△

## 💡 実際の活用例
- どのような文脈で登場するか
- 使われ方の具体例など

## 🏷️ 関連用語
`#タグ1` `#タグ2`
```

このテンプレートに沿って、Issueごとに用語生成を指示し、Kiroによる**自動実装→PR作成→レビュー→マージ**という流れで進めます。

---

## 🔥 特に強いのはこういう使い方！

| タスク内容 | Claude/Kiro得意度 | 補足 |
|------------|------------------|------|
| テンプレに沿った文章生成 | ◎（すごく得意） | 指定された構造でMarkdown生成が可能 |
| 構成パターンに従った大量生成 | ◎（繰り返し向き） | 同一フォーマットでの一括生成に強い |
| GitHub上でIssue → PRにする | ◯（設定すれば自動） | Claudeの出力をKiroがPR化可能 |
| 用語間リンクの自動付与 | △（要工夫） | 人間による追記または補助スクリプトが必要 |
| 内容の正しさチェック | △〜×（要人間確認） | LLMの出力には監修・レビューが必須 |

> このような特性から、本プロジェクトは「**人間がフォーマットとレビューを担い、Kiroが中間処理を自動化するハイブリッド構造**」で進めるのが最も適しています。

---

## 🚀 実行の流れ（想定）

1. **テンプレートをベースに用語をIssueとして投稿**
2. `@kiro` コマンドでPR作成を自動トリガー
3. Kiro（Claude）がMarkdown辞書ファイルを自動生成
4. PRとして提出、開発者が内容をレビュー・修正
5. マージして用語データベースを更新
